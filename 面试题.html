	11.说说浮动？
CSS样式中float就是浮动，可以标签上浮，提高半个层级，脱离文档流，失去原本的位置，向指定的方向移动，碰到父标签内容边缘或同样设置了float，都会将display属性值变为block。
	float样式表示浮动，浮动的出现初衷是为了实现文字包围图片，但是现在更多的使用浮动来进行网页布局。
     1>什么是浮动？
CSS样式中float就是浮动，可以标签上浮，提高半个层级，脱离文档流，失去原本的位置，向指定的方向移动，碰到父标签内容边缘或同样设置了float，都会将display属性值变为block。
2>float的作用
可以让设置了浮动的标签向上漂浮，然后向着指定的方向移动，直到碰到父标签或者同样设置了浮动的标签的边框才会停止。
3>.用float能干什么？
通常使用float属性进行页面布局，使标签（块标签）同行显示。
4>产生问题？
父标签失去高度（父级坍塌）。	                    
浮动的特点：
  1.向着指定方向移动，直到碰到父标签或其他浮动标签的边框才会停。
  2.可以使块标签同行显示，并且标签间没有间隙，也不用向inline-block一样选择对齐方式（基线）。
  3.设置了浮动的标签在父标签中是不占据位置的。
  4.当浮动标签因为层级标签提高遮挡了其他标签的时候，被遮挡标签的内容会显示在标签外。
  5.当父标签的宽度放不下浮动标签的时候，浮动标签会“换行”显示，浮动的换行与inline-block不同，浮动元素换行是参考的至于浮动方向上紧相邻的上一个标签为一行。
  6.因为浮动元素的特点，所以在使用浮动让块标签同行显示的时候，一定要注意标签的宽高，很容易出现参差不齐的情况

浮动带来的影响：父级坍塌，子标签设置浮动后在父标签中将不占据位置，那么原本应该由子标签高度撑开父标签的高度，父标签高度丢失。
解决办法：
  1，给父标签设置高度,缺点：写成固定高度，当标签内容增加或减少时，都需要重新设计高度，很不灵活。
  2.给父标签也设置浮动float属性，缺点：解决了当前父标签坍塌的问题，但是父标签的父标签又面临高度坍塌，需要一直向外层父标签添加浮动属性。
 3.给父标签添加over-flow属性，赋值为hidden，目前使用相对较多的方法。
over-flow属性：用来设置当子标签大小超出父标签范围，超出部分的显示方式。
  1.hidden，将超出父标签范围的部分隐藏
  2.scroll，将超出父标签范围的部分通过滚动条显示。
  3.auto，自动判断子标签内容是否超出，超出就通过滚动条显示。
  4.为父标签末尾添加一个空的div块标签，并设置clear属性并赋值both.（缺点：当页面标签较多的时候，会出现大量空白的块标签，增加代码阅读难度，不利于维护。）
    5.为父标签设置伪元素：为元素设置样式属性，通过after伪元素来消除浮动带来的影响，是现在主流的清除流动的解决办法。
#wrap::after{
	contennt“”；
	display：block;
	clear:both;
}

浮动可以使行标签设置宽高，并且支持上下的margin
注意：所有标签都可以浮动，但是浮动后的标签display属性都会被设置成block，并且在父标签中占据的空间不再是一行而是只有模盒型大小；不管浮动之前的宽高是如何得来的，浮动后，只要没设置固定的宽高，都是由内容撑开。

	14.媒体查询？移动端viewport？
		媒体查询：用在响应式布局中；注意：媒体查询一定要写在标签样式的下面或者层叠样式表的最下面才会起作用
		@media only/not mediaType and (条件){
			符合条件时候需要显示的样式
		}
		（1）@media：固定写法
		（2）修饰词：
			   only：表示只有/仅当后面条件成立的时候
			   not：表示除了后面条件以外的其他情况
		（3）mediaType：表示媒体类型
				1.screen：表示手机/平板/电脑的屏幕
				2.print：表示打印机或打印机预览
		（4）and：表示连接词，将媒体类型和条件做并列，一个媒体查询中可以出现多个连接词
		（5）条件：可以设置的有：
				width：表示屏幕宽度
				height：表示屏幕高度
				可以加前缀：min/max
		（6）当条件和媒体类型都符合的时候显示的样式

		移动端：viewport：可以理解成移动设备上浏览器的视窗（虚拟视窗），viewport视窗大小可以大于移动设备屏幕尺寸，也可以小于移动设备屏幕尺寸，可以自定义设置，通常情况下会设置为与移动设备屏幕同宽
		常用属性：
			1.width：用来设置浏览器虚拟视窗的宽度，常用值：device-width，表示将浏览器视窗宽度设置为设备屏幕宽度
			2.initial-scale：用来设置页面初次加载时默认的缩放比例，需要一个数值，不带单位
			3.maximum-scale：用来设置页面可被用户放大的最大比例
			4.minimum-scale：用来设置页面可被用户缩小的最小比例
			5.user-scalable：用来设置页面是否允许被用户缩放，设置为yes允许，设置为no不允许，一旦设置了不允许，上面maximum-scale和minimum-scale都会失去作用

	17.JS中的作用域？

		2）局部作用域：只能在局部声明的地方使用，当全局变量和局部变量重名时，存在就近原则。

	//作用域：简单理解为起作用的区域
//  函数内和函数外，函数外的代码会随着网页在浏览器加载自动的从上而下逐行执行，而函数内代码不会自动执行。
//  函数外：全局作用域。函数内：局部作用域（需要事件触发或者手动创建）
//   全局作用域内声明的变量称为全局变量，局部作用域声明的变量称为局部变量
//  全局变量，可以在脚本的任何位置访问使用；
而局部变量只能在自己所处的局部作用域访问使用。
//   (局部变量在函数运行结束后会被自动销毁)
//  当全局变量和局部变量重名时，存在就近原则。
//  JS存在变量提升的现象，会将变量的声明提升到当前所在作用域的最顶端

	28.什么是事件冒泡？什么是事件捕获？
		答：事件流的流向（执行顺序）：
1.事件冒泡（从里向外）
 从当前具体的元素开始，依次想外开始，依次向外执行父元素及祖先元素的事件处理程序。
 dom0级的事件绑定方式只支持事件冒泡.

2.事件捕获（从外向里）
 从最外层的祖先元素（window）开始，到当前具体的触发事件的元素，依次执行对应的事件处理程序。
 如果要实现事件捕获的效果，要使用dom2级的事件绑定方式。
(事件)
 addEventListener（attachEvent）的第三个参数决定事件流的方向，默认false（冒泡），设置成true代表捕获

如果在一个事件流的流向中同时存在事件捕获和事件冒泡，先由外向里按顺序执行所有的捕获，再由里向外，按循序执行所有的冒泡（先从里向外，再从外向里）

    出于浏览器的同源策略限制。同源策略（Sameoriginpolicy）是一种约定，它是浏览器最核心也最基本的安全功能，如果缺少了同源策略，则浏览器的正常功能可能都会受到影响。可以说Web是构建在同源策略基础之上的，浏览器只是针对同源策略的一种实现。同源策略会阻止一个域的javascript脚本和另外一个域的内容进行交互。所谓同源（即指在同一个域）就是两个页面具有相同的协议（protocol），主机（host）和端口号（port）

    什么是跨域?
	当一个请求url的协议、域名、端口三者之间任意一个与当前页面url不同即为跨域

	40.post和get请求的区别？如何选择？
		答：（1）post更安全（不会作为url的一部分，不会被缓存、保存在服务器日志、以及浏览器浏览记录中）
			（2）post发送的数据更大（get有url长度限制）
			（3）post能发送更多的数据类型（get只能发送ASCII字符）
			（4）post比get慢
			（5）post用于修改和写入数据，get一般用于搜索排序和筛选之类的操作（淘宝，支付宝的搜索查询都是get提交），目的是资源的获取，读取数据

			get比post请求快的原因：
				1）post请求包含更多的请求头
				2）post在真正接受数据之前会将请求头发给服务器进行确认，才会真正发送数据
				3）get会将数据缓存起来而post不会
				4）post不能进行管道化传输

			div.ontouchstart = function(event){
				touch = event.touches[0];
				touch.clientX    触摸点在浏览器窗口中的横坐标
				touch.pageX      触摸点在页面中的横坐标
				touch.screenX    触摸点在屏幕中的横坐标
			}

			div.ontouched = function(event){
				touch = event.touches[0];
				touch.clientX    触摸点在浏览器窗口中的横坐标
				touch.pageX      触摸点在页面中的横坐标
				touch.screenX    触摸点在屏幕中的横坐标
			}
